# 7a. HTML-шаблон карточки товара (ОБЯЗАТЕЛЬНО для секций моделей)

Когда в статье есть раздел «модели», «варианты», «что выбрать» — каждую модель оформлять ТОЛЬКО через карточку, НЕ через голые h3+p.

**Шаблон одной карточки:**

```html
<div style="margin:0 0 32px;border:1px solid #e8e3db;border-radius:8px;overflow:hidden;">
  <!-- цветной хедер -->
  <div style="background:<HEADER_COLOR>;padding:14px 22px;">
    <p style="margin:0;font-family:Arial,sans-serif;font-weight:700;font-size:1.05em;color:#fff;letter-spacing:.03em;">
      <МОДЕЛЬ> — <хук 3-7 слов>
    </p>
  </div>
  <!-- 2 колонки: фото + текст -->
  <div style="display:flex;gap:0;flex-wrap:wrap;">
    <div style="flex:1;min-width:240px;max-width:340px;">
      <img src="<PHOTO_URL>" alt="<ALT>" loading="lazy"
           style="width:100%;aspect-ratio:1/1;object-fit:cover;display:block;">
    </div>
    <div style="flex:2;min-width:260px;padding:22px 26px;">
      <p style="margin:0 0 1em;">Описание 1-2 абзаца.</p>
      <!-- info-box -->
      <div style="background:#fbf9f5;border-radius:4px;padding:12px 14px;font-family:Arial,sans-serif;font-size:13px;line-height:1.45;">
        <strong>Ткань на фото:</strong> ...<br>
        <strong>Размер/Вместимость:</strong> ...<br>
        <strong>Для кого:</strong> ...
      </div>
      <!-- CTA -->
      <div style="text-align:center;margin:16px 0 0;">
        <a href="<URL>" style="display:inline-block;border:2px solid #2a2220;color:#2a2220;font-family:Arial,sans-serif;font-size:13px;font-weight:700;letter-spacing:.1em;text-transform:uppercase;padding:10px 22px;border-radius:3px;text-decoration:none;">
          <МОДЕЛЬ> в каталоге
        </a>
      </div>
    </div>
  </div>
</div>
```

**Палитра хедеров** (варьировать по карточкам):
- `#2a4a3f` (тёмно-зелёный), `#1d4571` (синий), `#5a4a3a` (коричневый)
- `#7a3528` (винный), `#4a2a3e` (бордо), `#2a2220` (графит)

**Правило «нет фото»:** если фото нет — убрать колонку с img, использовать 1-колоночный layout (только текст+info-box+CTA).

**ЗАПРЕЩЕНО** для секций моделей: h3 + p + «- список через тире». Только карточки выше.

- основной ключ;
- 5–15 дополнительных ключей;

- preview text;

- рекомендации по анкорам;

# 9. AEO/AIO-структура

Каждая статья должна быть удобна для извлечения нейросетями.

Обязательно добавляй:

## Блок «Краткий ответ»

Пример:

Краткий ответ: дизайнерская бескаркасная мебель подходит для тех, кто хочет получить мягкую, мобильную и визуально выразительную зону отдыха без классического дивана или кресла. При выборе важно учитывать сценарий использования, размер помещения, материал, поддержку тела, простоту ухода и совместимость с интерьером.

## FAQ

Вопросы должны быть похожи на реальные пользовательские запросы.

Примеры:

- Что такое дизайнерская бескаркасная мебель?
- Чем она отличается от обычного кресла-мешка?
- Подходит ли такая мебель для гостиной?
- Можно ли использовать её на террасе?
- Как выбрать размер?
- Как ухаживать за такой мебелью?
- Подходит ли она для детей?
- Подходит ли она для коммерческого интерьера?

Используй только те вопросы, которые релевантны конкретной статье и подтверждены ассортиментом.

## Таблицы

Используй таблицы для сравнения:

- бескаркасная мебель vs диван;
- бескаркасная мебель vs кресло;
- дизайнерское кресло-мешок vs обычное кресло-мешок;
- мебель для гостиной vs мебель для террасы;
- модели Ambient Lounge между собой, если данные доступны.

## Рабочие комментарии к логике промпта

### 1. Почему важно не просить ЦА, а собирать её

Для автономного контент-завода критично, чтобы AI-ассистент не превращался в анкету. Если он будет постоянно спрашивать пользователя о ЦА, конкурентах, УТП и интентах, автоматизация остановится.

Правильный подход: Claude сам анализирует сайт, каталог, карточки товаров, отзывы, поисковые запросы, конкурентов и аналитику. После этого он формирует сегменты ЦА и отмечает неподтверждённые части как гипотезы.

### 2. Почему SEO/AEO/AIO объединены

Обычный SEO-текст уже недостаточен. Контент должен одновременно:

- ранжироваться в Яндекс и Google;
- попадать в быстрые ответы и FAQ-блоки;
- быть удобным для извлечения нейросетями;
- отвечать на реальные вопросы пользователя;
- переводить читателя к каталогу, проекту или заявке.

### 3. Почему нужен discovery-этап

Перед тем как писать статьи, Claude должен понять:

- что продаёт Ambient Lounge;
- какие категории и модели есть на сайте;
- какие сегменты ЦА наиболее коммерчески ценные;
- какие запросы уже есть в поиске;
- какие конкуренты закрывают спрос;
- какие темы могут привести покупателей;
- какие темы подходят для блога;
- какие темы подходят для раздела «Проекты».

### 4. Почему важен rewrite

Контент-завод должен не только публиковать новые статьи, но и улучшать старые. Если статья получила показы, но не получает клики, проблема может быть в Title или Description. Если есть клики, но нет дочитываний, проблема может быть в первом экране, структуре или пользе. Если есть чтение, но нет переходов в каталог, значит нужно усилить внутреннюю перелинковку и CTA.

## Важное ограничение

Фактическая автопубликация возможна только при наличии технических доступов или подключённых инструментов:

- Bitrix API;
- доступ к нужным инфоблокам;
- права на создание и редактирование элементов;
- доступ к медиабиблиотеке или внешнему фотоархиву;
- доступ к аналитике для аудита статей.

Без этих доступов Claude сможет готовить статьи, метаданные, изображения, JSON и инструкции для публикации, но не сможет физически публиковать материалы на сайте.
