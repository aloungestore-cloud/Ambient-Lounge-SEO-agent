# CLAUDE.md — Правила для AI-агентов: SEO-AIO Agent

Этот файл читается Claude Code и AI-агентами при работе с проектом.

## Проект

**SEO-AIO Agent** — автономный контент-завод для сайта ambientlounge.ru.  
Задача: ежедневный поиск тем → написание статей → SEO/AEO/AIO-оптимизация → публикация в Bitrix/АСПРО → аудит эффективности → rewrite.

## Правила работы агентов

### 1. Source of truth для конфигурации

Все URL сайта, статусы доступов к API и аналитике — только в `docs/project_config.md`.  
Не хранить доступы в коде или других файлах.

### 2. Автономность

Не спрашивать у пользователя то, что можно собрать самостоятельно:
- ЦА — извлекать из сайта, отзывов, поисковых запросов
- Ассортимент — парсить из предоставленных URL каталога
- Конкуренты — находить через поиск и анализ
- Интенты — определять по запросам и содержанию страниц

Неизвестное помечать: `[ГИПОТЕЗА]` или `[ТРЕБУЕТ_ПОДТВЕРЖДЕНИЯ]`.  
Не останавливать работу из-за отсутствия данных.

### 3. Журналирование

После каждой публикации или аудита обновлять `docs/audit_log.md`.  
После добавления темы в план — обновлять `docs/content_calendar.md`.

### 4. Модерация перед публикацией

Перед финальной выдачей любой статьи выполнять чек-лист из раздела 19 мастер-промпта.  
Статья без AEO/AIO-структуры, FAQ и метаданных — не публикуется.

### 5. Формат вывода

Каждая сессия должна заканчиваться:
- обновлённой таблицей тем;
- готовыми статьями с метаданными;
- JSON-пакетами для Bitrix;
- обновлением `docs/audit_log.md`;
- датой следующей проверки опубликованных статей.

### 6. Конфиденциальность

Не раскрывать в текстах статей: себестоимость, чистую прибыль, внутренние скидки, данные CRM.

### 7. Что нельзя трогать без явного указания

- `CLAUDE.md` — этот файл
- `templates/` — изменять только по явной команде
