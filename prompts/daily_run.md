# Daily Run — Ежедневный запуск контент-завода

Короткая команда для запуска ежедневного цикла.  
Скопируй и передай Claude вместе с URL сайта.

---

## Команда запуска

```text
Запусти ежедневный цикл SEO/AEO/AIO-контент-завода для ambientlounge.ru.

Прочитай файлы проекта в порядке:
1. docs/project_config.md — URL сайта и статусы доступов
2. ambientlounge_claude_seo_aeo_aio_content_factory_prompt.md — полный мастер-промпт
3. docs/channel_specs.md — HTML-архитектура и шаблон карточки товара (§1.5 — ОБЯЗАТЕЛЬНО)
4. templates/aida_article_template.md — КАНОНИЧЕСКИЙ шаблон AIDA (ОБЯЗАТЕЛЬНО, каждая статья строго по нему)
5. docs/audit_log.md — что опубликовано, что ждёт аудита
6. docs/content_calendar.md — текущая очередь тем

Затем выполни полный рабочий цикл:
1. Собери данные по сайту из предоставленных URL.
2. Найди 15–30 тем, оцени по формуле, выбери 5–10 лучших.
3. Напиши статьи СТРОГО по шаблону templates/aida_article_template.md (AIDA).
4. Подготовь метаданные, изображения, внутреннюю перелинковку.
5. Сформируй JSON-пакеты для Bitrix/АСПРО.
6. Обнови docs/audit_log.md и docs/content_calendar.md.
7. Укажи дату следующей проверки опубликованных статей.

КРИТИЧЕСКИ ВАЖНО для каждой статьи сайта:
- HTML строго по aida_article_template.md — порядок блоков фиксирован
- product cards с кнопками «в каталоге» в каждой карточке (≥3 карточек)
- Schema.org LD+JSON (Article + FAQPage) В КОНЦЕ detail_text — без исключений
- Нуль плейсхолдеров [ТРЕБУЕТ_ПОДТВЕРЖДЕНИЯ] в финальном HTML

Если каких-то данных нет, не останавливайся:
- используй обоснованные гипотезы с пометкой [ГИПОТЕЗА] только в черновике
- критичное уточни или исключи из статьи — не публикуй неподтверждённые факты
```

---

## Команда аудита (раз в 7 дней)

```text
Запусти аудит опубликованных статей для ambientlounge.ru.

Прочитай docs/audit_log.md и docs/project_config.md.

Для каждой статьи со статусом «audited» или «indexed»:
1. Проанализируй метрики (GSC / Метрика — если доступны, иначе отметь).
2. Сравни с бенчмарками из project_config.md.
3. Определи статьи, требующие rewrite.
4. Составь план переработки с приоритетами.
5. Обнови docs/audit_log.md.
```

## Команда rewrite

```text
Перепиши статью: [URL]

Причина rewrite: [указать проблему]
Доступные метрики: [показы / клики / CTR / позиция / время на странице]

Прочитай оригинальную статью, определи слабые места, перепиши с сохранением URL slug.
Обнови JSON-пакет для Bitrix. Запиши изменения в docs/audit_log.md.
```

## Команда discovery (первый запуск)

```text
Проведи discovery-анализ проекта Ambient Lounge.

Прочитай:
1. docs/project_config.md — URL сайта
2. ambientlounge_claude_seo_aeo_aio_content_factory_prompt.md — раздел 2 (Discovery)

Собери и структурируй:
- ассортимент по предоставленным URL
- сегменты ЦА
- позиционирование и УТП
- конкурентов по группам
- контент-кластеры
- первичную семантику

Результат сохрани в docs/content_calendar.md (кластеры) и выдай отчёт.
```
